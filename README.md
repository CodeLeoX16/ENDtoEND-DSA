## üéØ What You'll Learn

### Data Structures
- **Arrays & Strings**
  - Basic operations and manipulations
  - Two-pointer techniques
  - Sliding window problems
  
- **Linked Lists**
  - Singly and Doubly Linked Lists
  - Circular Linked Lists
  - Fast and Slow pointer techniques
  
- **Stacks & Queues**
  - Implementation and applications
  - Monotonic stack/queue problems
  
- **Trees**
  - Binary Trees & Binary Search Trees
  - Tree traversals (Inorder, Preorder, Postorder, Level Order)
  - AVL Trees, Red-Black Trees
  
- **Graphs**
  - Graph representations
  - Graph traversals (BFS, DFS)
  - Advanced graph algorithms
  
- **Heaps**
  - Min Heap & Max Heap
  - Priority Queue applications
  
- **Hash Tables**
  - Hash functions
  - Collision resolution techniques
  
- **Advanced Data Structures**
  - Tries
  - Segment Trees
  - Fenwick Trees (Binary Indexed Tree)
  - Disjoint Set Union (DSU)

### Algorithms
- **Sorting Algorithms**
  - Bubble Sort, Selection Sort, Insertion Sort
  - Merge Sort, Quick Sort, Heap Sort
  - Counting Sort, Radix Sort
  
- **Searching Algorithms**
  - Linear Search, Binary Search
  - Binary Search variations
  
- **Recursion & Backtracking**
  - Understanding recursion
  - Backtracking patterns
  - N-Queens, Sudoku Solver
  
- **Dynamic Programming**
  - Memoization vs Tabulation
  - Classic DP problems
  - Advanced DP techniques
  
- **Greedy Algorithms**
  - Greedy choice property
  - Activity selection, Huffman coding
  
- **Graph Algorithms**
  - Shortest Path (Dijkstra, Bellman-Ford, Floyd-Warshall)
  - Minimum Spanning Tree (Kruskal, Prim)
  - Topological Sort
  - Strongly Connected Components
  
- **String Algorithms**
  - Pattern matching (KMP, Rabin-Karp)
  - Longest Common Subsequence
  - Edit Distance

## üö¶ How to Use This Repository

1. **Start from the Basics**: If you're new to DSA, begin with fundamental data structures like arrays and linked lists
2. **Follow the Playlist Order**: The topics are organized in a progressive manner
3. **Practice Regularly**: Consistency is key - try to solve at least one problem daily
4. **Understand, Don't Memorize**: Focus on understanding the concepts rather than memorizing solutions
5. **Implement from Scratch**: Try implementing each data structure and algorithm yourself
